DRR-AI Overview

Designed for Restricted-Use Data

Privacy and Security by Design

  • DRR-AI is designed to comply with restricted-use data license policies — your manuscript data is never retained, logged, or shared.
  • All reviews are conducted entirely in memory using an ephemeral, zero-retention architecture — nothing is written to disk or stored on servers.
  • Our system architecture enables researchers to audit their manuscripts securely and privately through the DRR-AI tool.
DRR-AI Component 1

AI Layer

Zero-Data-Retention API

Legally Binding Enterprise Agreement

DRR-AI uses a zero-data-retention API provided for commercial enterprise use. These enterprise APIs are governed by legally binding agreements stating that prompt data is not stored and is not used to train models.

Unlike consumer AI products, enterprise APIs carry contractual data protection guarantees — your manuscript content is processed and immediately discarded by the AI provider.
DRR-AI Component 2

Backend Layer

In-Memory File Handling

Session-Only Processing

DRR-AI parses Word and PDF files directly from the incoming web request buffer — files are never written to disk. Review results are displayed only within the active session. Once you leave the session, the results are no longer accessible.

What happens to your file

Read into memory → text extracted → sent to AI → result returned → memory cleared. No disk writes at any step.

What we log

Only non-identifying metadata: timestamp, framework used, token counts, and processing time. No file content, no text.
